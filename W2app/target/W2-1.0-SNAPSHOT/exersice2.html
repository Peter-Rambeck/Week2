Q:
Most import part of this exercise:
Explain (on paper) using both words and images how the Server can maintain state between subsequent calls even when the state is not transmitted from the client to server.
A:
# Caching on Cookie.
# HTTP is stateless. A cookie makes the server identify the state
# When the server place an ID cookie in the Session, it cache the req's made and data state of the app.
# As the Session req. the server for data, it includes the cookie ID, hence the server lookup the responses made/the data state
# The server maintains the state with it's memory and cookie ID number.
# A Session cookie's life is approx. 30min.
